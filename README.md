# 忆站 Elder Care Miniapp

一个面向老人记忆支持与家庭陪伴场景的微信小程序项目。

项目分为两个角色端：
- 老人端：查看家人、回忆、健康信息、留言板、生活教程、卡片游戏
- 家属端：维护老人资料、家庭成员、回忆、健康数据、家人留言、生活教程

项目基于微信云开发实现，业务主要集中在云函数 `yizhanService` 中。

## 项目定位

“忆站”希望解决的不是单一功能问题，而是老人日常生活中几个常见痛点：
- 容易忘记家人关系和家庭成员信息
- 回忆内容分散，难以形成有温度的记忆触发
- 健康数据缺少持续记录与提醒
- 家人陪伴无法及时、自然地传递
- 家电或日常物品使用步骤容易忘记

所以这个项目把“记忆支持、家庭陪伴、健康提醒、生活辅助”放进了一个统一的小程序中。

## 主要功能

### 老人端

- 登录后进入老人首页
- 首页默认展示“留言板”，支持查看：
  - 家人语音留言
  - 家人文字留言
  - 今日服药提醒
- 首页可切换查看“回忆”
- 首页底部拆分为两块：
  - `日常陪伴`：我的资料、看家人、今日健康
  - `时光乐园`：看回忆、生活小助手、卡片游戏
- 家庭成员查看
- 历史回忆浏览
- 今日健康概览与异常提醒
- 生活小助手教程查看
- 卡片配对记忆游戏
- 人脸识别查看家人资料卡

### 家属端

- 维护老人基础资料
- 绑定老人账号
- 管理家庭成员
- 上传和编辑回忆
- 为回忆设置完整日期，支持“那年今日”
- 区分“本人回忆”和“家人回忆”
- 录入病史、用药、健康测量数据
- 查看血压/血糖趋势与异常提醒
- 代录健康数据
- 发送家人留言：
  - 语音留言
  - 纯文字留言
  - 语音 + 文字备注
- 管理生活小助手教程：
  - 一个教程可包含多个步骤
  - 每个步骤支持单独图片 + 单独文字说明
  - 可选上传演示视频

## 项目亮点

- 老人端和家属端职责明确，界面复杂度有明显区分
- 留言板和服药提醒合并展示，减少老人查找成本
- “那年今日”使用完整日期而不是只有年份
- 生活小助手采用“分步骤图文教程”，更贴近日常使用场景
- 家属端可持续维护老人健康记录，老人端只展示最关键结果
- 预留并接入了腾讯云人脸识别能力

## 核心数据集合

云函数中使用的集合如下：

- `users`
  - 老人和家属账号信息
- `persons`
  - 家庭成员信息
- `memories`
  - 家属维护的回忆内容
- `healthRecords`
  - 病史、用药、每日健康测量
- `elder_uploads`
  - 老人端上传的图片/文字内容
- `memory_pairs`
  - 卡片配对游戏素材
- `voice_messages`
  - 家人留言板数据
- `life_guides`
  - 生活小助手教程数据

## 业务架构说明

### 1. 登录与角色流

- 登录页支持选择：
  - `elder`
  - `family`
- 登录成功后：
  - 老人跳转到 `/pages/elder/home`
  - 家属跳转到 `/pages/family/home`
- 本地使用 `wx.setStorageSync` 保存：
  - `role`
  - `token`
  - `userId`
  - `elderId`（家属绑定老人后使用）

### 2. 接口调用方式

- 小程序统一通过 `miniprogram/api/user.js` 调用云函数
- 所有业务动作通过 `action` 分发到 `yizhanService`
- 家属端在已绑定老人后，会自动把 `elderId` 透传给云函数

### 3. 老人端首页

老人端首页目前是项目的核心中枢：

- 默认进入 `留言板`
- 可切换到 `回忆`
- 留言板中合并展示：
  - 未读家人留言
  - 服药提醒
- 下方再通过底部切换分成：
  - `日常陪伴`
  - `时光乐园`

这种结构的好处是老人不需要面对过多并列入口，而是先看到最重要的信息。

### 4. 健康模块设计

- 家属端：
  - 可代录血压、心率、血糖
  - 可维护病史和用药
  - 可查看趋势和异常提醒
- 老人端：
  - 只保留今日健康概览和提醒

这是典型的“复杂管理留给家属，简单结果留给老人”的设计。

### 5. 生活小助手设计

- 家属端创建教程时，一个教程可以有多个步骤
- 每个步骤包含：
  - 图片
  - 文字说明
- 可选上传一个演示视频
- 老人端查看时，按步骤顺序展示，每一步都是独立图文

这个模块适合：
- 电饭煲怎么煮饭
- 微波炉怎么加热
- 热水器怎么开
- 药盒怎么取药
- 遥控器怎么调台

## 人脸识别说明

项目已接入腾讯云人脸识别调用逻辑，主要能力包括：

- 为家庭成员同步标准人脸照
- 识别老人端拍摄的人脸
- 命中后返回家庭成员资料卡
- 未命中显示“未知人士”

需要在云函数环境变量中配置：

```env
TENCENTCLOUD_SECRET_ID=你的SecretId
TENCENTCLOUD_SECRET_KEY=你的SecretKey
TENCENTCLOUD_REGION=ap-shanghai
FACE_SCORE_THRESHOLD=85
```

如果不配置以上变量，人脸识别功能无法正常工作，但项目其他功能不受影响。

## 安装与运行

### 1. 安装本地依赖

根目录依赖：

```bash
npm install
```

云函数依赖如果未自动安装，请在 `cloudfunctions/yizhanService` 下安装：

```bash
cd cloudfunctions/yizhanService
npm install
```

### 2. 使用微信开发者工具打开项目

- 导入项目根目录
- 确认 `project.config.json` 中的小程序配置正常
- 打开云开发能力

### 3. 配置云环境

- 修改 [app.js](/c:/Users/LIQIANYUE/OneDrive/文档/大三课外/计算机设计大赛/1/elder-care-miniapp/miniprogram/app.js) 中的 `env`
- 或改成你自己的云开发环境 ID

### 4. 部署云函数

需要部署：

- `cloudfunctions/yizhanService`

每次改动云函数后都要重新部署，否则前端会继续调用旧逻辑。

### 5. 初始化测试流程

建议按这个顺序测试：

1. 登录家属端
2. 绑定老人
3. 录入老人资料
4. 录入家庭成员
5. 新增回忆
6. 发送留言
7. 录入健康数据和服药信息
8. 新增生活小助手教程
9. 切到老人端查看展示结果
